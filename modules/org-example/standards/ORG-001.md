# ORG-001: Use approved HTTP client library

## Standard

Python services MUST use `requests` or `httpx` for HTTP calls. `urllib` and `http.client` are not approved for production use.

## Rationale

- Approved libraries have consistent error handling, retry logic, and timeout defaults
- `urllib` has footguns (no timeout by default, verbose API, poor error messages)
- Consistent HTTP library across the org simplifies debugging and shared middleware

## What good looks like

- All HTTP calls use `requests.get()` / `httpx.Client()`
- Shared retry/timeout configuration via a wrapper or middleware
- No `import urllib` or `from http.client` in production code

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Regex for `import urllib` or `from http.client` in .py files | Every PR |

**Primary enforcement: Automated.** The VS Code extension and CI checker both flag urllib/http.client imports.
