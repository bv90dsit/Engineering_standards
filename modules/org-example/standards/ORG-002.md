# ORG-002: Internal APIs must use .internal.gov.uk domain

## Standard

Internal service-to-service API calls MUST use `.internal.gov.uk` domain names, not raw IP addresses or `localhost` in configuration.

## Rationale

- DNS-based service discovery enables load balancing and failover
- TLS certificate validation requires a valid hostname
- Raw IPs break when infrastructure changes; DNS provides indirection
- Consistent naming convention makes firewall rules auditable

## What good looks like

- All service URLs in config use `https://service-name.internal.gov.uk`
- No hardcoded IP addresses in application config or environment variables
- `localhost` only in local development config (never in production/staging)

```yaml
# GOOD: environment config (e.g., Kubernetes ConfigMap or .env)
PAYMENT_SERVICE_URL=https://payment-service.internal.gov.uk
USER_SERVICE_URL=https://user-service.internal.gov.uk
NOTIFICATION_SERVICE_URL=https://notification-service.internal.gov.uk
```

```python
# GOOD: service call using DNS name
import os, requests
response = requests.get(
    f"{os.environ['USER_SERVICE_URL']}/api/v1/users/{user_id}",
    timeout=10,
)
```

## What bad looks like

```yaml
# BAD: hardcoded IP addresses
PAYMENT_SERVICE_URL=http://10.0.3.45:8080
USER_SERVICE_URL=http://192.168.1.100:3000

# BAD: localhost in non-dev config
NOTIFICATION_SERVICE_URL=http://localhost:8081
```

```python
# BAD: hardcoded IP in application code
response = requests.get("http://10.0.3.45:8080/api/payments")
```

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Regex for hardcoded IPs or non-.internal.gov.uk URLs in config files | Every PR |

**Primary enforcement: Automated.** The VS Code extension and CI checker flag hardcoded IPs and localhost in non-dev config files.

## Quick reference

```yaml
# All service URLs must use .internal.gov.uk
SERVICE_URL=https://service-name.internal.gov.uk
# Never: http://10.x.x.x or http://localhost in production config
```
