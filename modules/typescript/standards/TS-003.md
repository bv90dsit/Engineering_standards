# TS-003: API responses must be typed

## Standard

All API responses from `fetch`, `axios`, or similar HTTP clients SHOULD be validated against a defined type. Do not rely on implicit `any` from `.json()`. Use runtime validation (Zod, io-ts) or at minimum an interface with type assertion.

## Rationale

- `response.json()` returns `any` by default, silently disabling type safety for all downstream code
- APIs change without notice — fields get renamed, removed, or change type
- Runtime validation catches mismatches at the boundary rather than causing cryptic failures deep in business logic
- Typed API responses serve as living documentation of the contract between services

## What good looks like

- Runtime validation with Zod, io-ts, or similar at API boundaries

```typescript
// Best - runtime validation with Zod
import { z } from 'zod';

const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
  email: z.string().email(),
});

type User = z.infer<typeof UserSchema>;

async function fetchUser(id: number): Promise<User> {
  const response = await fetch(`/api/users/${id}`);
  const data = await response.json();
  return UserSchema.parse(data); // Throws if shape doesn't match
}

// Acceptable - interface with type assertion (no runtime check)
interface UserResponse {
  id: number;
  name: string;
  email: string;
}

async function fetchUser(id: number): Promise<UserResponse> {
  const response = await fetch(`/api/users/${id}`);
  return (await response.json()) as UserResponse;
}

// Bad - untyped response
async function fetchUser(id: number) {
  const response = await fetch(`/api/users/${id}`);
  return response.json(); // Returns any
}
```

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | Reviewer verifies that all API call sites have typed responses | PR review |
| **Peer review** | Reviewer checks for Zod/io-ts usage on critical paths | PR review |

**Primary enforcement: Peer review.** This standard is difficult to fully automate. Reviewers check that fetch/axios calls have return types and ideally runtime validation on external API boundaries.
