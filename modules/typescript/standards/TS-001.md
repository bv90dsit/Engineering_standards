# TS-001: Strict mode enabled in tsconfig.json

## Standard

Every TypeScript project MUST have `"strict": true` in its `tsconfig.json` (or all individual strict sub-flags explicitly enabled). This applies to all new and existing projects.

## Rationale

- Without strict mode, TypeScript is barely better than JavaScript — it allows implicit `any`, unchecked `null`/`undefined`, and loose function typing
- Strict mode enables `strictNullChecks`, `noImplicitAny`, `strictFunctionTypes`, `strictBindCallApply`, `strictPropertyInitialization`, and `noImplicitThis`
- Catching type errors at compile time prevents entire classes of runtime bugs
- Consistent strict configuration across projects makes code portable between repositories

## What good looks like

```json
// tsconfig.json
{
  "compilerOptions": {
    "strict": true,
    "target": "ES2020",
    "module": "commonjs"
  }
}
```

Alternatively, all strict sub-flags enabled individually:

```json
{
  "compilerOptions": {
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "strictBindCallApply": true,
    "strictPropertyInitialization": true,
    "noImplicitThis": true,
    "alwaysStrict": true
  }
}
```

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | `tsconfig.json` contains `"strict": true` or all strict sub-flags enabled | Every PR |
| **Automated (VS Code)** | Rules engine flags `"strict": false` in any tsconfig file | On save |

**Primary enforcement: Automated.** A CI check parses `tsconfig.json` and fails the build if strict mode is not enabled. The VS Code extension rule flags any explicit `"strict": false` setting.
