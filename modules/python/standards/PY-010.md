# PY-010: Use pathlib for file operations

## Standard

Python code SHOULD use `pathlib.Path` instead of `os.path` for file operations. pathlib is cross-platform, more readable, and less error-prone.

## Rationale

- `pathlib.Path` provides an object-oriented interface that is more intuitive than string-based `os.path` functions
- Path objects handle OS-specific separators automatically, reducing cross-platform bugs
- Method chaining (e.g., `path.parent / "sibling" / "file.txt"`) is more readable than nested `os.path.join` calls
- `pathlib` was added in Python 3.4 (PEP 428) and is the recommended approach in modern Python
- Common patterns like checking existence, reading/writing files, and iterating directories are more concise with pathlib
- Reduces import clutter â€” no need for `import os, os.path, glob`

## What good looks like

```python
from pathlib import Path

# Building paths
config_dir = Path.home() / ".config" / "myapp"
data_file = config_dir / "data.json"

# Checking existence
if data_file.exists():
    content = data_file.read_text()

# Creating directories
config_dir.mkdir(parents=True, exist_ok=True)

# Iterating files
for py_file in Path("src").rglob("*.py"):
    print(py_file.name)

# Getting parts of a path
stem = data_file.stem        # "data"
suffix = data_file.suffix    # ".json"
parent = data_file.parent    # Path(".config/myapp")
```

## What bad looks like

```python
import os
import os.path
import glob

# BAD: os.path for joining
config_dir = os.path.join(os.path.expanduser("~"), ".config", "myapp")
data_file = os.path.join(config_dir, "data.json")

# BAD: os.path for checking existence
if os.path.exists(data_file):
    with open(data_file, "r") as f:
        content = f.read()

# BAD: os.makedirs instead of Path.mkdir
os.makedirs(config_dir, exist_ok=True)

# BAD: glob.glob instead of Path.rglob
for py_file in glob.glob("src/**/*.py", recursive=True):
    print(os.path.basename(py_file))

# BAD: os.path for path components
stem = os.path.splitext(os.path.basename(data_file))[0]
suffix = os.path.splitext(data_file)[1]
parent = os.path.dirname(data_file)
```

## Enforcement

| Mechanism | What is checked | When |

| **Peer review** | Reviewer flags new `os.path` usage and suggests pathlib equivalent | PR review |

**Primary enforcement: Peer review.** Reviewers encourage pathlib adoption in new code. Existing `os.path` usage need not be migrated unless the file is being substantially modified.

## Quick reference

| os.path | pathlib equivalent |
|---------|-------------------|
| `os.path.join(a, b)` | `Path(a) / b` |
| `os.path.exists(p)` | `Path(p).exists()` |
| `os.path.isfile(p)` | `Path(p).is_file()` |
| `os.path.isdir(p)` | `Path(p).is_dir()` |
| `os.path.basename(p)` | `Path(p).name` |
| `os.path.dirname(p)` | `Path(p).parent` |
| `os.path.splitext(p)` | `Path(p).stem` / `.suffix` |
| `open(p).read()` | `Path(p).read_text()` |
| `os.makedirs(p)` | `Path(p).mkdir(parents=True)` |
| `glob.glob("**/*.py")` | `Path(".").rglob("*.py")` |
