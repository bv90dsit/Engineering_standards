# PY-009: Pin dependencies with a lock file

## Standard

Python projects MUST commit a lock file (`requirements.txt` with pinned versions, `poetry.lock`, or `Pipfile.lock`). Un-pinned dependencies allow supply chain attacks and break reproducibility.

## Rationale

- Without pinned versions, any `pip install` may pull in a new, potentially malicious package version without explicit review
- Reproducibility is impossible when dependency versions float — builds become non-deterministic across environments
- Supply chain attacks frequently exploit version ranges: a compromised patch release is automatically consumed
- Lock files create an auditable record of exactly which packages (and transitive dependencies) are deployed
- Pinning enables automated vulnerability scanning tools to accurately identify affected versions

## What good looks like

```text
# requirements.txt — all versions pinned (output of pip freeze)
django==4.2.11
psycopg2-binary==2.9.9
celery==5.3.6
redis==5.0.3
```

```toml
# pyproject.toml with poetry — poetry.lock committed to VCS
[tool.poetry.dependencies]
python = "^3.11"
django = "^4.2"
```

```bash
# Generating / updating the lock file
pip freeze > requirements.txt
# or
poetry lock
# or
pipenv lock
```

## What bad looks like

```text
# BAD: requirements.txt with unpinned or loosely pinned versions
django
celery>=5.0
redis
requests>=2.0,<3.0
```

```text
# BAD: .gitignore excludes the lock file
# .gitignore
poetry.lock
Pipfile.lock
```

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | CI step verifies existence of at least one lock file (requirements.txt with `==` pins, poetry.lock, or Pipfile.lock) | Every PR |

**Primary enforcement: Automated.** A CI check verifies that the repository contains a committed lock file. The check fails if no recognised lock file is present or if `requirements.txt` contains unpinned entries (missing `==` specifiers).

## Quick reference

```bash
# pip
pip freeze > requirements.txt

# poetry
poetry lock

# pipenv
pipenv lock
```
