# PY-008: Django DEBUG must be False in production config

## Standard

Django's `DEBUG` setting MUST be set to `False` in production configuration files. Debug mode MUST NOT be enabled in any environment accessible to end users.

## Rationale

- `DEBUG = True` exposes full stack traces including source code, local variables, and file paths to any user who triggers an error
- Debug mode displays all executed SQL queries in error pages, potentially revealing sensitive data
- The debug toolbar and error pages expose Django settings, including database credentials and secret keys
- Debug mode disables some security features (e.g. host header validation is relaxed via ALLOWED_HOSTS)
- Memory usage increases in debug mode because Django stores every executed query

## What good looks like

```python
# settings/production.py
import os

DEBUG = False
# Or better, from environment:
DEBUG = os.environ.get("DJANGO_DEBUG", "False").lower() == "true"

ALLOWED_HOSTS = [".example.com"]
```

## What bad looks like

```python
# BAD: DEBUG enabled in production settings
# settings/production.py
DEBUG = True

# BAD: DEBUG defaulting to True
DEBUG = os.environ.get("DJANGO_DEBUG", True)
```

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Grep for `DEBUG = True` or `DEBUG=True` in settings files | Every PR |

**Primary enforcement: Automated.** A CI check scans production settings files for `DEBUG = True`. Local development and test settings files (e.g. `settings/local.py`, `settings/dev.py`) are excluded from this check.

## Quick reference

```python
import os
DEBUG = os.environ.get("DJANGO_DEBUG", "False").lower() == "true"
ALLOWED_HOSTS = [".example.com"]
```
