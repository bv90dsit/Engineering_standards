# SEC-001: HTTPS everywhere

## Standard

All services MUST use HTTPS for every connection, with no HTTP fallback. TLS 1.2 is the minimum; TLS 1.3 SHOULD be preferred.

## Rationale

- Protects data in transit from interception and tampering
- Required by NCSC guidance and the Service Standard
- Enables HTTP/2 and modern browser security features (HSTS, CSP)

## What good looks like

- TLS termination at the load balancer or CDN with valid certificates (minimum TLS 1.2, prefer 1.3)
- HSTS headers with `max-age=31536000; includeSubDomains; preload`
- Services on the HSTS preload list where possible (submit at https://hstspreload.org)
- No mixed content warnings
- Certificate renewal is automated (e.g. Let's Encrypt, AWS ACM)
- HTTP requests return 301 redirect to HTTPS (not served over HTTP)

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | No `http://` URLs in application source code (excluding tests and tooling) | Every PR |
| **Periodic audit** | HSTS headers configured, certificates valid and auto-renewed, TLS version ≥ 1.2 | Service assessment and quarterly security review |

**Primary enforcement: Automated + Periodic audit.** The pipeline catches hardcoded HTTP URLs in code. Infrastructure-level TLS configuration (HSTS, cert renewal, protocol version) is verified during security assessment using tools like `ssllabs.com` or `testssl.sh`.

## Quick reference

**HSTS header (set in your application or load balancer):**

```
Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
```

**Nginx TLS config:**

```nginx
server {
    listen 443 ssl http2;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384';
    ssl_prefer_server_ciphers off;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    return 301 https://$host$request_uri;
}
```

**AWS ALB/CloudFront** — TLS 1.2+ is the default policy. Use `TLSv1.2_2021` or newer security policy.

**Test your TLS configuration:**

```bash
# Quick check from command line
curl -sI https://your-service.gov.uk | grep -i strict-transport

# Full TLS audit (install testssl.sh: brew install testssl)
testssl --protocols --headers https://your-service.gov.uk

# Or use SSL Labs (browser): https://www.ssllabs.com/ssltest/
```

**CI check for hardcoded HTTP URLs:**

```yaml
- name: No HTTP URLs in source
  run: |
    if grep -rn 'http://' src/ --include='*.ts' --include='*.js' --include='*.py' \
       | grep -v 'http://localhost' | grep -v 'http://127.0.0.1' | grep -v '# noqa'; then
      echo "::error::Found hardcoded http:// URLs in source code"
      exit 1
    fi
```
