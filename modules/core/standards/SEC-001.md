# SEC-001: HTTPS everywhere

## Standard

All services MUST use HTTPS for every connection, with no HTTP fallback. TLS 1.2 is the minimum; TLS 1.3 SHOULD be preferred.

## Rationale

- Protects data in transit from interception and tampering
- Required by NCSC guidance and the Service Standard
- Enables HTTP/2 and modern browser security features (HSTS, CSP)

## What good looks like

- TLS termination at the load balancer or CDN with valid certificates
- HSTS headers with a minimum max-age of 1 year
- Services on the HSTS preload list where possible
- No mixed content warnings
- Certificate renewal is automated (e.g. Let's Encrypt, AWS ACM)

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | No `http://` URLs in application source code (excluding tests and tooling) | Every PR |
| **Periodic audit** | HSTS headers configured, certificates valid and auto-renewed, TLS version ≥ 1.2 | Service assessment and quarterly security review |

**Primary enforcement: Automated + Periodic audit.** The pipeline catches hardcoded HTTP URLs in code. Infrastructure-level TLS configuration (HSTS, cert renewal, protocol version) is verified during security assessment using tools like `ssllabs.com` or `testssl.sh`.
