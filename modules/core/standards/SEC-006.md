# SEC-006: Container image scanning

## Standard

Container images SHOULD be scanned for vulnerabilities before deployment. Base images SHOULD be minimal (distroless/alpine) and regularly rebuilt. Images SHOULD be signed and verified.

## Rationale

- Container images contain OS-level dependencies with their own vulnerability lifecycle, separate from application dependencies
- Base images accumulate CVEs over time if not rebuilt regularly
- Unsigned images risk supply chain attacks via image substitution
- Minimal images reduce attack surface and scan noise

## What good looks like

- Trivy (or Grype/Snyk) scanning integrated into CI, blocking on CRITICAL/HIGH findings with `--exit-code 1`
- Minimal base images (e.g., `gcr.io/distroless/static-debian12`, `alpine:3.20`) rebuilt at least weekly via scheduled pipeline
- No root user in containers (`USER nonroot:nonroot` in Dockerfile)
- Image signing with `cosign sign` and verification with `cosign verify` on pull
- Private registry (e.g., GitHub Container Registry, ECR, Artifact Registry) with built-in vulnerability scanning enabled
- Clear policy: CRITICAL CVEs remediated within 48 hours, HIGH within 7 days, or documented risk acceptance

## What bad looks like

- Full OS base images (ubuntu:latest) never rebuilt
- Containers running as root
- No scanning of images before deployment
- Images pulled from untrusted public registries without verification
- Ignoring scan results with no documented risk acceptance

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Trivy or equivalent scans container image on build; blocks deployment on critical/high findings | Every build producing a container image |

**Primary enforcement: Automated.** Container scanning runs as part of the CI pipeline. Critical and high-severity findings block the image from being pushed to the registry or deployed to production.

## Quick reference

**Trivy scan (block on CRITICAL/HIGH):**

```bash
trivy image --severity CRITICAL,HIGH --exit-code 1 --ignore-unfixed my-registry/my-app:latest
```

**Minimal Dockerfile:**

```dockerfile
FROM gcr.io/distroless/static-debian12:nonroot
COPY --chown=nonroot:nonroot app /app
USER nonroot:nonroot
ENTRYPOINT ["/app"]
```

**Cosign signing and verification:**

```bash
# Sign (CI signs after build+push)
cosign sign --key cosign.key my-registry/my-app@sha256:<digest>

# Verify (runtime policy or deploy gate)
cosign verify --key cosign.pub my-registry/my-app@sha256:<digest>
```

**GitHub Actions CI snippet:**

```yaml
- name: Scan container image
  uses: aquasecurity/trivy-action@master
  with:
    image-ref: ${{ env.IMAGE }}
    severity: CRITICAL,HIGH
    exit-code: '1'
    ignore-unfixed: true
```
