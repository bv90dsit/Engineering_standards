# EMG-003: AI outputs validated before user-facing use

## Standard

AI-generated content or decisions that will be shown to users or acted upon MUST be validated before use. Validation may be human review, automated quality checks, or both depending on risk. Services MUST NOT present AI output as authoritative without validation.

## Rationale

- AI systems hallucinate, reflect training biases, and can produce harmful output
- Government services carry authority and must be accurate
- Users may not know output is AI-generated

## What good looks like

- Human-in-the-loop for high-stakes decisions — e.g. caseworker reviews every AI-drafted decision letter before sending
- Automated checks for factual claims — citation verification against source database, cross-referencing dates and figures
- Confidence thresholds below which output is rejected or escalated — e.g. model confidence < 0.85 triggers human review queue
- Output monitoring for drift — weekly metrics on rejection rate, confidence distribution, and user-reported errors
- Clear labelling when content shown to users was AI-generated or AI-assisted
- Fallback path defined when AI system is unavailable (graceful degradation, not silent failure)

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | Architecture review confirms validation approach | Design phase |
| **Automated (CI/CD)** | Quality gate tests in CI for AI output pipelines | PR check and deployment |

**Primary enforcement: Peer review + Automated.** Architecture review confirms validation approach; quality gate tests in CI for AI output pipelines.

## Quick reference

**Validation pipeline pseudocode (confidence threshold):**
```python
CONFIDENCE_THRESHOLD = 0.85
TOXIC_THRESHOLD = 0.1

def validate_ai_output(response):
    if response.confidence < CONFIDENCE_THRESHOLD:
        return route_to_human_review(response)
    if response.toxicity_score > TOXIC_THRESHOLD:
        return reject(response, reason="toxicity")
    if not verify_citations(response.citations):
        return route_to_human_review(response, reason="unverified_citations")
    return approve(response)
```

**CI quality gate for AI output (GitHub Actions example):**
```yaml
- name: Validate AI output quality
  run: |
    python scripts/validate_ai_outputs.py \
      --min-confidence 0.85 \
      --max-toxicity 0.1 \
      --check-citations \
      --fail-on-violation
```

**Human-in-the-loop workflow:**
```
Request --> [AI Model] --> [Automated Checks] --pass--> [Published]
                                |
                                |--fail/low confidence
                                v
                          [Human Review Queue]
                                |
                        --------+--------
                        |               |
                    [Approve]      [Reject/Edit]
                        |               |
                  [Published]    [Feedback to model team]
```
