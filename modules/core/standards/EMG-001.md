# EMG-001: AI-generated code must be reviewed as any other contribution

## Standard

Code produced by AI assistants (Copilot, Claude, Cursor, etc.) MUST go through the same review, testing, and approval process as human-written code. The engineer who submits AI-generated code is accountable for its correctness, security, and maintainability.

## Rationale

- AI tools can introduce subtle bugs, security flaws, and licence-incompatible code
- Accountability must remain with a named human, not delegated to a tool
- Consistent quality bar regardless of how code was produced

## What good looks like

- AI-generated code is committed via normal pull request flow with peer review
- Reviewers are told (or can see from tooling) when AI was used in generation
- Automated security scanning (SAST, SCA) runs on all code regardless of origin
- Engineers can explain and justify every line they submit, including AI-generated lines
- AI tools are configured to respect repository .gitignore and secrets patterns

## What bad looks like

- Pasting AI output directly into production without review
- Disabling linters or tests because "the AI said it works"
- Using AI tools on OFFICIAL-SENSITIVE or above codebases without approved tooling
- Treating AI suggestions as authoritative for security-sensitive logic

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | AI-generated code goes through normal PR flow; reviewer can identify and question AI-generated sections | Every PR |
| **Ways of working** | Team agreement on approved AI tools; logging/attribution of AI usage; no AI on OFFICIAL-SENSITIVE+ without approved tooling | Documented in team charter; reviewed at onboarding |

**Primary enforcement: Peer review + Ways of working.** This standard is fundamentally about human accountability. The pipeline can verify branch protection requires review (a prerequisite), but whether AI code was *actually reviewed critically* is a team-culture question enforced through ways of working.
