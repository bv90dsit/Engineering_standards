# ACC-003: Content written in plain English

## Standard

User-facing content SHOULD be written in plain English at a reading age of 9 (Flesch-Kincaid grade level ≤ 9). Technical jargon SHOULD be avoided or explained.

## Rationale

- GOV.UK research shows the average UK reading age is 9 — writing above this level excludes a significant portion of the population
- Even specialist audiences prefer plain language — it is faster to read and reduces misinterpretation
- Users under stress (dealing with benefits, legal matters, health issues) have reduced reading comprehension
- Plain English supports users with dyslexia, cognitive disabilities, and those for whom English is a second language
- Clear language reduces support queries and user errors

## What good looks like

- Content tested against readability tools and achieves Flesch-Kincaid grade level ≤ 9
- Short sentences (average 15-20 words) and short paragraphs (2-4 sentences)
- Active voice preferred over passive voice
- Common words used instead of formal alternatives (see replacements table below)
- Technical terms explained on first use or linked to a glossary
- Bullet points and numbered lists used for instructions and grouped information
- Content reviewed by a content designer or second pair of eyes for clarity
- User research confirms content is understood by the target audience

## What bad looks like

- Dense paragraphs of formal language with sentences over 30 words
- Jargon used without explanation (e.g. "submit your DPIA to the IAO for triage")
- Passive voice obscuring who does what (e.g. "The form must be completed" vs "Complete the form")
- Latin phrases and legalese in user-facing content (e.g. "inter alia", "notwithstanding")
- Content written by subject-matter experts without content design review
- No readability testing performed

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | Content reviewed for plain English during PR review; readability score checked for user-facing text | Every PR with user-facing content changes |

**Primary enforcement: Peer review.** Content changes are reviewed for plain English compliance during code/content review. Automated readability scoring may supplement but is not the primary mechanism due to limitations of automated tools with technical content.

## Quick reference

**Readability tools:**

| Tool | Usage | Notes |
|------|-------|-------|
| [Hemingway App](https://hemingwayapp.com/) | Paste text, check grade level | Free web tool, highlights complex sentences |
| readability-cli | `npx readability-cli check content.md` | CLI tool for CI integration |
| [GOV.UK Content Design guidance](https://www.gov.uk/guidance/content-design/writing-for-gov-uk) | Reference guide | Canonical source for government plain English |

**readability-cli usage:**

```bash
# Install and run against a content file
npx readability-cli check src/content/apply-for-licence.md

# In CI pipeline (fail if grade level > 9)
npx readability-cli check --max-grade 9 src/content/**/*.md
```

**Common replacements (jargon to plain English):**

| Instead of | Use |
|-----------|-----|
| utilise | use |
| commence | start |
| terminate | end / stop |
| facilitate | help |
| in order to | to |
| prior to | before |
| subsequently | then / after |
| regarding | about |
| in the event of | if |
| it is necessary to | you need to / you must |
| at this point in time | now |
| a number of | some |
| in respect of | about / for |
| notwithstanding | even though / despite |
| pursuant to | under / following |
| endeavour | try |
| ascertain | find out |
| sufficient | enough |
| in excess of | more than |
| on a daily basis | every day |

**Sentence structure tips:**

```
BAD:  "In the event that your application is not successful, you will
       be notified in writing within 28 days of the decision being made."

GOOD: "If your application is unsuccessful, we will write to you within
       28 days."

BAD:  "Applicants are required to furnish documentation pertaining to
       their eligibility prior to the commencement of the assessment."

GOOD: "Send us documents that show you are eligible before your
       assessment starts."
```
