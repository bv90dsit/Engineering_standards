# SEC-002: Dependency vulnerability scanning

## Standard

All repositories MUST have automated dependency vulnerability scanning enabled. Critical and high severity vulnerabilities MUST be remediated within 7 days of disclosure. Scanning MUST run in CI and on a scheduled basis.

## Rationale

- Third-party dependencies are the most common attack vector (OWASP A06:2021)
- Government services are high-value targets; known vulnerabilities are actively exploited
- Automated scanning catches issues that manual review cannot at scale

## What good looks like

- Dependabot, Snyk, Trivy, or equivalent configured and active
- Alerts triaged within 48 hours of notification
- Critical/high vulnerabilities patched or mitigated within 7 days
- Scanning runs in CI (fail on critical) and on a weekly schedule
- A documented process for when a vulnerability cannot be patched (compensating controls)
- Lock files committed to pin dependency versions

## What bad looks like

- No scanning configured
- Hundreds of unacknowledged vulnerability alerts
- Dependencies pinned to ancient versions with no update strategy
- "We'll fix it when we have time" on critical severity findings

## Remediation timeline

| Severity | Maximum time to remediate |
|----------|--------------------------|
| Critical | 7 days |
| High | 14 days |
| Medium | 30 days |
| Low | Next planned maintenance |

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Dependabot, Snyk, Renovate, or Trivy configuration file exists in repo | Every PR |
| **Automated (scheduled)** | Scanner runs on a weekly schedule and alerts are triaged | Continuous — scanner platform reports |

**Primary enforcement: Automated.** The compliance checker verifies scanner config is present. Alert triage SLA (7 days for critical) is monitored via the scanning platform's own reporting dashboard.

## Quick reference

**Dependabot config** (`.github/dependabot.yml`):

```yaml
version: 2
updates:
  # npm dependencies
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
    open-pull-requests-limit: 10
    labels:
      - "dependencies"
    commit-message:
      prefix: "chore(deps):"

  # Docker base images
  - package-ecosystem: "docker"
    directory: "/"
    schedule:
      interval: "weekly"

  # GitHub Actions
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"

  # Python (pip)
  # - package-ecosystem: "pip"
  #   directory: "/"
  #   schedule:
  #     interval: "weekly"
```

**Trivy in CI** (GitHub Actions):

```yaml
- name: Run Trivy vulnerability scanner
  uses: aquasecurity/trivy-action@master
  with:
    scan-type: 'fs'
    scan-ref: '.'
    severity: 'CRITICAL,HIGH'
    exit-code: '1'            # Fail the build on critical/high
    ignore-unfixed: true
```

**Renovate** alternative (for more control than Dependabot) — add `renovate.json` to repo root:

```json
{
  "$schema": "https://docs.renovatebot.com/renovate-schema.json",
  "extends": ["config:recommended"],
  "schedule": ["every weekend"],
  "vulnerabilityAlerts": { "enabled": true },
  "labels": ["dependencies"]
}
```

**Enable GitHub secret/dependency scanning (org-wide):**

```bash
# Enable Dependabot alerts + security updates for a repo
gh api repos/{owner}/{repo}/vulnerability-alerts --method PUT
gh api repos/{owner}/{repo} --method PATCH -f security_and_analysis[dependabot_security_updates][status]=enabled
```

**Manual check for vulnerabilities (local):**

```bash
# npm
npm audit --audit-level=high

# Python
pip-audit

# Docker image
trivy image your-image:latest --severity HIGH,CRITICAL
```
