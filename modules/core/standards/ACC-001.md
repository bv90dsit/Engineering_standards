# ACC-001: WCAG 2.2 AA compliance

## Standard

All public-facing services MUST meet WCAG 2.2 Level AA. This is a legal requirement under the Public Sector Bodies (Websites and Mobile Applications) Accessibility Regulations 2018.

## Rationale

- Legal obligation under the Public Sector Bodies Accessibility Regulations 2018
- 1 in 5 people in the UK have a disability
- Exclusion from digital services = exclusion from public services

## What good looks like

- Automated accessibility testing in CI (axe-core, pa11y) — e.g. `pa11y-ci` runs against all routes on every PR, failing the build on any AA violation
- Manual testing with assistive technologies — at minimum: NVDA or VoiceOver walkthrough of the core journey before go-live; periodic testing with Dragon NaturallySpeaking and ZoomText
- Accessibility statement published and linked from every page footer (use the GOV.UK sample template as a starting point)
- Team includes accessibility testing in definition of done — "Story not done until pa11y passes and manual keyboard walkthrough completed"
- Colour contrast ratios meet AA minimums (4.5:1 normal text, 3:1 large text) — verified using browser DevTools or a tool like Colour Contrast Analyser
- New components tested with axe DevTools extension during development, not just in CI

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | axe-core or pa11y checks fail on new accessibility violations | Every PR |
| **Periodic audit** | Full WCAG 2.2 AA audit, assistive technology testing, user testing with disabled users | Beta assessment, annual review |

**Primary enforcement: Automated + Periodic audit.** CI catches regressions via axe-core/pa11y. Comprehensive accessibility audits are conducted at beta assessment and annually, including user testing with disabled users.

## Quick reference

**pa11y-ci config** (`.pa11yci` in repo root):
```json
{
  "defaults": {
    "standard": "WCAG2AA",
    "timeout": 10000,
    "wait": 1000
  },
  "urls": [
    "http://localhost:3000/",
    "http://localhost:3000/start",
    "http://localhost:3000/confirm"
  ]
}
```

**package.json script:**
```json
"scripts": {
  "test:a11y": "pa11y-ci"
}
```

**axe-core in Jest (jest-axe):**
```js
const { axe, toHaveNoViolations } = require('jest-axe');
expect.extend(toHaveNoViolations);

it('has no accessibility violations', async () => {
  const { container } = render(<MyComponent />);
  const results = await axe(container);
  expect(results).toHaveNoViolations();
});
```

**axe-core in Cypress (cypress-axe):**
```js
it('passes accessibility checks', () => {
  cy.visit('/start');
  cy.injectAxe();
  cy.checkA11y(null, { runOnly: ['wcag2a', 'wcag2aa'] });
});
```
