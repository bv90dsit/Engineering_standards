# DAT-002: Personal data has a retention policy

## Standard

All personal data MUST have a defined and documented retention period. Data MUST be deleted or anonymised when the retention period expires. Retention policies MUST be technically enforced (not just documented).

## Rationale

- UK GDPR Article 5(1)(e) — storage limitation principle requires data not be kept longer than necessary
- ICO enforcement action risk for non-compliance
- Data breaches are worse when you hold data you don't need
- Reduces storage costs and attack surface

## What good looks like

- Retention schedule documented in DPIA with explicit periods (e.g. "application data deleted 6 months after decision")
- Automated deletion/anonymisation jobs running on schedule, not relying on manual intervention
- TTL on database records where appropriate (e.g. DynamoDB TTL attribute, or scheduled SQL deletes)
- Audit trail of deletions showing what was removed, when, and by which job
- Regular review (at least annual) of what is held vs what is needed
- Clear justification for each retention period tied back to a lawful basis

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Periodic audit** | Retention schedule exists and matches DPIA; automated enforcement is running; deletion audit trail reviewed | DPIA review, ICO accountability framework assessment |

**Primary enforcement: Periodic audit.** Evidence of automated retention enforcement is reviewed as part of the DPIA process and ICO accountability framework compliance.

## Quick reference

**Example retention schedule:**

| Data category | Retention period | Trigger | Action |
|---------------|-----------------|---------|--------|
| Application form data | 6 months | Decision date | Delete |
| Successful applicant records | 6 years | Employment end | Anonymise |
| Audit logs | 2 years | Creation date | Delete |
| Marketing consent | Until withdrawn | Withdrawal | Delete |

**Django management command (run via cron):**

```python
# app/management/commands/enforce_retention.py
from django.core.management.base import BaseCommand
from datetime import timedelta
from django.utils import timezone
from myapp.models import Application

class Command(BaseCommand):
    def handle(self, **options):
        cutoff = timezone.now() - timedelta(days=180)
        deleted, _ = Application.objects.filter(
            decision_date__lt=cutoff
        ).delete()
        self.stdout.write(f"Deleted {deleted} expired applications")
```

```bash
# crontab entry — run nightly at 02:00
0 2 * * * cd /srv/app && python manage.py enforce_retention >> /var/log/retention.log 2>&1
```

**PostgreSQL scheduled deletion (pg_cron):**

```sql
SELECT cron.schedule(
  'delete-expired-applications',
  '0 2 * * *',
  $$DELETE FROM applications WHERE decision_date < now() - interval '180 days'$$
);
```
