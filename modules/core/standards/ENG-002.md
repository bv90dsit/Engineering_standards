# ENG-002: Version control for all source code

## Standard

All source code, infrastructure-as-code, and configuration MUST be stored in a version control system (Git). Repositories MUST have branch protection on the default branch requiring at least one peer review before merge.

## Rationale

- Enables collaboration, auditability, and rollback
- Branch protection prevents untested or unreviewed code reaching production
- Git history provides a complete audit trail for compliance

## What good looks like

- Every service, library, and infrastructure definition lives in Git
- Default branch (main) is protected: requires PR, at least 1 approval, passing CI
- Commit messages are meaningful and reference ticket IDs where applicable
- Repositories are organised under a team or programme-level GitHub organisation
- Stale branches are cleaned up regularly

## What bad looks like

- Code shared via email, shared drives, or USB sticks
- Direct pushes to main with no review
- Repositories with no branch protection rules
- "Final_v2_FINAL_really_final.zip" in SharePoint

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Repository is a git repo with a remote configured | Every PR |
| **Peer review** | Branch protection enforced — PR required, at least 1 approval, CI must pass | Continuous (GitHub/GitLab settings) |

**Primary enforcement: Automated + Peer review.** Pipeline verifies git remote exists. Branch protection rules enforce the review requirement — these are configured once per repo and checked via GitHub API at audit.

## Quick reference

**Set up branch protection (GitHub CLI):**

```bash
# Require PR reviews + status checks on 'main'
gh api repos/{owner}/{repo}/branches/main/protection \
  --method PUT \
  --input - <<'EOF'
{
  "required_status_checks": {
    "strict": true,
    "contexts": ["ci"]
  },
  "enforce_admins": true,
  "required_pull_request_reviews": {
    "required_approving_review_count": 1,
    "dismiss_stale_reviews": true
  },
  "restrictions": null
}
EOF
```

**Branch protection via GitHub UI:**
Settings > Branches > Add rule for `main`:
- [x] Require a pull request before merging (1 approval minimum)
- [x] Require status checks to pass before merging
- [x] Require branches to be up to date before merging
- [x] Do not allow bypassing the above settings

**Commit message convention (Conventional Commits):**

```
<type>(<scope>): <short summary>  [JIRA-123]

Types: feat, fix, docs, style, refactor, test, chore
Example: feat(auth): add MFA support for admin users  [PLAT-456]
```

**Clean up stale branches:**

```bash
# Delete merged remote branches (dry-run first)
git branch -r --merged origin/main | grep -v main | sed 's/origin\///' | xargs -I {} echo "would delete: {}"
# Then run for real:
git branch -r --merged origin/main | grep -v main | sed 's/origin\///' | xargs -I {} git push origin --delete {}
```
