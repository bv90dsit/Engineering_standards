# ENG-003: Continuous integration and automated testing

## Standard

Every repository MUST have a CI pipeline that runs automatically on every pull request. The pipeline MUST include automated tests. Code MUST NOT merge to the default branch with failing tests.

## Rationale

- Catches defects early, before they reach production
- Reduces the cost of fixing bugs (exponentially cheaper when caught in CI vs production)
- Gives teams confidence to deploy frequently (see OPS-002)
- Prevents "works on my machine" failures

## What good looks like

- CI runs on every PR and blocks merge on failure
- Test suite covers critical paths (unit + integration at minimum)
- CI completes in under 10 minutes for fast feedback
- Flaky tests are tracked and fixed, not ignored
- Code coverage is measured (target: 80%+ for new code, trending upward)
- Linting and static analysis run as part of CI

## What bad looks like

- Manual testing as the only quality gate
- CI that is perpetually red and ignored
- Tests that only run on a schedule (nightly) rather than per-PR
- Merge buttons that bypass failing checks

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | At least one CI workflow file exists in `.github/workflows/`, `.gitlab-ci.yml`, or `Jenkinsfile` | Every PR |

**Primary enforcement: Automated.** The compliance checker verifies a CI config is present. The *quality* of the test suite (coverage, flakiness) is reviewed during periodic audit.

## Quick reference

**Minimal GitHub Actions CI workflow** (`.github/workflows/ci.yml`):

```yaml
name: CI
on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up runtime
        uses: actions/setup-node@v4   # or setup-python, setup-java, etc.
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Run tests
        run: npm test

      - name: Check coverage threshold
        run: npm test -- --coverage --coverageThreshold='{"global":{"lines":80}}'
```

**GitLab CI equivalent** (`.gitlab-ci.yml`):

```yaml
stages: [test]

test:
  stage: test
  image: node:20
  script:
    - npm ci
    - npm run lint
    - npm test -- --coverage
  rules:
    - if: $CI_PIPELINE_SOURCE == "merge_request_event"
    - if: $CI_COMMIT_BRANCH == $CI_DEFAULT_BRANCH
```

**Key settings to enforce:**
- Branch protection: require CI to pass before merge (see ENG-002)
- Set a 10-minute timeout on CI jobs to detect hung tests
- Add `--ci` / `--frozen-lockfile` flags to prevent unexpected dependency changes
