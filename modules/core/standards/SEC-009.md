# SEC-009: CORS explicitly configured

## Standard

APIs MUST NOT use wildcard (*) CORS origins in production. Allowed origins MUST be explicitly listed. Credentials-mode requests MUST specify exact origins.

## Rationale

- Wildcard CORS origins allow any website to make cross-origin requests to your API, enabling data exfiltration
- Credentials (cookies, authorization headers) cannot be sent with wildcard origins per the spec, but misconfigured servers may reflect the Origin header, bypassing this protection
- Explicit origin lists ensure only trusted frontends can access APIs cross-origin
- Misconfigured CORS is a common attack vector for stealing user data

## What good looks like

- Allowed origins explicitly listed in configuration, loaded from environment variables (not hard-coded)
- Separate CORS configuration per environment (dev allows localhost, production allows only known domains)
- `Access-Control-Allow-Credentials: true` only set when exact origins are specified
- Preflight (OPTIONS) responses cached appropriately with `Access-Control-Max-Age`
- Only required HTTP methods and headers listed in `Access-Control-Allow-Methods` and `Access-Control-Allow-Headers`
- CORS configuration tested in integration tests — verify that unknown origins are rejected
- Internal APIs not exposed to browsers do not set CORS headers at all

## What bad looks like

- `Access-Control-Allow-Origin: *` in production for APIs that handle user data
- Reflecting the incoming `Origin` header directly into `Access-Control-Allow-Origin` without validation
- Overly permissive regex matching (e.g. `*.example.com` matching `attacker-example.com`)
- `Access-Control-Allow-Credentials: true` combined with a reflected or wildcard origin
- No CORS configuration at all on APIs consumed by browser clients (relying on browser same-origin policy alone)

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | CORS configuration reviewed for wildcard usage and origin validation logic | Every PR modifying API or CORS config |
| **Automated** | Linter/CI check scans for `Access-Control-Allow-Origin: *` in production config; integration test sends request from unknown origin and asserts rejection | Every pipeline run |

**Primary enforcement: Peer review + Automated.** Code review catches logic errors in origin validation; CI scans prevent wildcard origins from reaching production.

## Quick reference

**Express cors middleware:**

```javascript
const cors = require('cors');

const allowedOrigins = process.env.ALLOWED_ORIGINS.split(',');
// e.g. ALLOWED_ORIGINS=https://www.example.gov.uk,https://admin.example.gov.uk

const corsOptions = {
  origin: (origin, callback) => {
    // Allow requests with no origin (server-to-server, curl)
    if (!origin) return callback(null, true);
    if (allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
  allowedHeaders: ['Content-Type', 'Authorization'],
  maxAge: 86400, // Cache preflight for 24 hours
};

app.use(cors(corsOptions));
```

**Spring Boot @CrossOrigin (controller-level):**

```java
@RestController
@CrossOrigin(
    origins = {"https://www.example.gov.uk", "https://admin.example.gov.uk"},
    allowCredentials = "true",
    methods = {RequestMethod.GET, RequestMethod.POST},
    maxAge = 86400
)
@RequestMapping("/api/v1/applications")
public class ApplicationController {
    // ...
}
```

**Spring Boot global CORS configuration:**

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Value("${app.cors.allowed-origins}")
    private String[] allowedOrigins;

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins(allowedOrigins)
            .allowedMethods("GET", "POST", "PUT", "DELETE")
            .allowCredentials(true)
            .maxAge(86400);
    }
}
```

**Nginx CORS headers:**

```nginx
location /api/ {
    # Only allow specific origins
    set $cors_origin "";
    if ($http_origin ~* "^https://(www|admin)\.example\.gov\.uk$") {
        set $cors_origin $http_origin;
    }

    add_header Access-Control-Allow-Origin $cors_origin always;
    add_header Access-Control-Allow-Credentials true always;
    add_header Access-Control-Allow-Methods "GET, POST, PUT, DELETE" always;
    add_header Access-Control-Allow-Headers "Content-Type, Authorization" always;
    add_header Access-Control-Max-Age 86400 always;

    # Handle preflight
    if ($request_method = OPTIONS) {
        return 204;
    }
}
```
