# OPS-005: Change failure rate tracked (DORA)

## Standard

Teams SHOULD track change failure rate (percentage of deployments that cause a degradation requiring remediation). Target: below 15% (DORA "high performer" threshold). Teams SHOULD review and act on this metric monthly.

## Rationale

- Change failure rate correlates with software delivery performance and organizational outcomes
- Tracking it creates feedback loop for process improvement

## What good looks like

- Metric tracked per service, reviewed monthly using the formula:
  `change_failure_rate = (failed_deployments + rollbacks + hotfixes) / total_deployments * 100`
- Trend visible on team dashboard with monthly and quarterly views
- DORA benchmark targets clearly communicated: Elite < 5%, High < 15%, Medium < 30%
- High failure rate triggers process review (not blame) — e.g. "why did 4 of our last 20 deploys require rollback?"
- Rollback count, hotfix count, and incident-causing deploys all included in calculation
- Metric broken down by deployment type (feature, bugfix, config change) to identify patterns

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Periodic audit** | Quarterly DORA review | Quarterly |
| **Ways of working** | Metric on team dashboard, discussed at retros | Ongoing — documented in team charter |

**Primary enforcement: Periodic audit + Ways of working.** Quarterly DORA review; metric on team dashboard, discussed at retros.

## Quick reference

**Change failure rate formula:**

```
CFR = (failed_deployments + rollbacks + hotfixes) / total_deployments * 100

Example: 3 rollbacks + 1 hotfix out of 40 deploys = (4 / 40) * 100 = 10% CFR
```

**DORA tracking tools:**

| Tool | Model | Notes |
|------|-------|-------|
| [Sleuth](https://www.sleuth.io/) | Git + deploy integration | Auto-detects failures from rollbacks and incidents |
| [LinearB](https://linearb.io/) | Git + project management | Correlates CFR with PR size and cycle time |
| [Four Keys](https://github.com/dora-team/fourkeys) | Open source (GCP) | Google's reference DORA implementation |

**Example dashboard metric query (Prometheus + deployment labels):**

```promql
# Change failure rate over last 30 days
(
  sum(deployments_total{result="failure"} or deployments_total{result="rollback"}) 
  / sum(deployments_total)
) * 100
```
