# SEC-004: Input validation and output encoding

## Standard

All user input MUST be validated against an allowlist. All output rendered in HTML, SQL, or shell contexts MUST be properly encoded/escaped. Parameterised queries MUST be used for all database access.

## Rationale

- Injection is consistently in the OWASP Top 10
- XSS and SQL injection remain the most exploited web vulnerabilities
- Proper input validation and output encoding prevent entire classes of attacks
- Defence in depth — validate at boundaries, encode at output

## What good looks like

- Input validated at system boundaries (type, length, format, allowlist)
- Parameterised queries used for all database access (never string concatenation for SQL)
- Template engines configured with auto-escaping enabled
- SAST tools running in CI (Semgrep, CodeQL) to catch injection patterns
- Content-Security-Policy headers deployed to mitigate XSS impact
- Context-aware encoding (HTML entity encoding, URL encoding, JavaScript escaping)

## What bad looks like

- String concatenation to build SQL queries
- User input rendered directly in HTML without escaping
- `innerHTML` or `dangerouslySetInnerHTML` with unsanitised content
- Blocklist-based validation instead of allowlist
- Validation only on the client side

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | Reviewers check for raw SQL, unescaped output, and missing input validation | Every PR |
| **Automated (CI/CD)** | SAST tools (Semgrep, CodeQL) flag injection patterns | Every PR |

**Primary enforcement: Peer review + Automated.** SAST tools catch known injection patterns automatically. Peer reviewers verify that input validation and output encoding are applied correctly, catching cases that static analysis may miss.

## Quick reference

**Parameterised queries (do this, not string concatenation):**

```python
# Python (psycopg2) - CORRECT
cursor.execute("SELECT * FROM users WHERE email = %s", (user_email,))

# Python - WRONG (SQL injection)
cursor.execute(f"SELECT * FROM users WHERE email = '{user_email}'")
```

```javascript
// Node.js (pg) - CORRECT
const result = await client.query('SELECT * FROM users WHERE email = $1', [userEmail]);

// Node.js - WRONG
const result = await client.query(`SELECT * FROM users WHERE email = '${userEmail}'`);
```

```java
// Java (JDBC) - CORRECT
PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE email = ?");
ps.setString(1, userEmail);
```

**Input validation (example with Zod for TypeScript):**

```typescript
import { z } from 'zod';

const CreateUserSchema = z.object({
  email: z.string().email().max(254),
  name: z.string().min(1).max(100).regex(/^[a-zA-Z\s'-]+$/),
  age: z.number().int().min(13).max(150),
});

// Validate at the boundary (controller/route handler)
const parsed = CreateUserSchema.safeParse(req.body);
if (!parsed.success) {
  return res.status(400).json({ errors: parsed.error.issues });
}
```

**Template engine auto-escaping (ensure enabled):**

```python
# Jinja2 - enable auto-escaping
from jinja2 import Environment, select_autoescape
env = Environment(autoescape=select_autoescape(['html', 'xml']))
```

```javascript
// Nunjucks - auto-escaping on by default
const nunjucks = require('nunjucks');
nunjucks.configure('views', { autoescape: true });
```

**Content-Security-Policy header:**

```
Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self'; img-src 'self' data:; frame-ancestors 'none';
```

**Semgrep in CI** (catches injection patterns):

```yaml
- name: Run Semgrep SAST
  uses: semgrep/semgrep-action@v1
  with:
    config: >-
      p/owasp-top-ten
      p/javascript
      p/python
```

**CodeQL setup** (`.github/workflows/codeql.yml`):

```yaml
name: CodeQL
on:
  pull_request:
    branches: [main]
  schedule:
    - cron: '0 6 * * 1'   # Weekly Monday 6am

jobs:
  analyze:
    runs-on: ubuntu-latest
    permissions:
      security-events: write
    strategy:
      matrix:
        language: ['javascript', 'python']   # add your languages
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: ${{ matrix.language }}
      - uses: github/codeql-action/analyze@v3
```
