# SEC-003: No secrets in source code

## Standard

Secrets (API keys, passwords, tokens, private keys, connection strings) MUST NOT be committed to source code repositories. Repositories MUST have pre-commit hooks or CI checks to prevent secrets from being pushed.

## Rationale

- Number one source of government security incidents
- Once in git history, secrets are nearly impossible to fully remove
- Bots scan public repositories within seconds of a push
- A single leaked credential can compromise entire environments

## What good looks like

- `detect-secrets` or `truffleHog` configured in pre-commit hooks AND CI
- Secrets stored in a vault (AWS Secrets Manager, Azure Key Vault, HashiCorp Vault)
- `.env` files listed in `.gitignore` (with a `.env.example` showing required keys without values)
- Secrets rotated immediately on suspected exposure
- GitHub push protection and secret scanning enabled on all repositories
- README documents how to obtain secrets for local development without committing them

## What bad looks like

- API keys hardcoded in source files
- Credentials in configuration files committed to the repository
- `.env` files checked into version control
- Secrets passed as plaintext command-line arguments in CI scripts

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Automated (CI/CD)** | Secret scanning with git-secrets, truffleHog, or GitHub secret scanning; fails PR if secrets detected | Every PR and push |

**Primary enforcement: Automated.** Pre-commit hooks catch secrets before they enter the repository. CI scanning provides a second layer, failing PRs that contain patterns matching known secret formats.

## Quick reference

**Pre-commit config** (`.pre-commit-config.yaml`):

```yaml
repos:
  - repo: https://github.com/Yelp/detect-secrets
    rev: v1.5.0
    hooks:
      - id: detect-secrets
        args: ['--baseline', '.secrets.baseline']
```

**Install and initialise:**

```bash
# Install pre-commit
pip install pre-commit

# Install the hooks into your repo
pre-commit install

# Generate initial baseline (marks existing false positives)
detect-secrets scan > .secrets.baseline

# Audit the baseline to confirm no real secrets
detect-secrets audit .secrets.baseline
```

**CI check** (GitHub Actions):

```yaml
- name: Detect secrets
  uses: Yelp/detect-secrets-action@v2
  with:
    baseline_file: .secrets.baseline
```

**Alternative: truffleHog in CI:**

```yaml
- name: TruffleHog secret scan
  uses: trufflesecurity/trufflehog@main
  with:
    extra_args: --only-verified
```

**Essential `.gitignore` entries:**

```gitignore
# Secrets and environment files
.env
.env.local
.env.*.local
*.pem
*.key
credentials.json
service-account.json
```

**`.env.example` template** (commit this so devs know what's needed):

```bash
# Copy to .env and fill in values from the team vault
DATABASE_URL=
API_KEY=
JWT_SECRET=
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
```

**If a secret is accidentally committed:**

```bash
# 1. Rotate the secret immediately (this is the priority)
# 2. Remove from history (if repo is public)
git filter-repo --invert-paths --path <file-with-secret>
# 3. Force push (coordinate with team)
git push --force-with-lease
# 4. Notify security team
# 5. Update .secrets.baseline
```

**Enable GitHub push protection (org-wide):**

```bash
gh api orgs/{org}/code-security/configurations --method POST \
  -f name="secret-scanning" \
  -f secret_scanning=enabled \
  -f secret_scanning_push_protection=enabled
```
