# JV-004: REST controllers must validate input with @Valid

## Standard

All `@RequestBody` parameters in REST controllers SHOULD be annotated with `@Valid` (or `@Validated`) to trigger Jakarta Bean Validation before the request reaches business logic.

## Rationale

- Without `@Valid`, malformed or malicious input passes directly to service/business logic layers
- Bean Validation annotations (`@NotNull`, `@Size`, `@Email`, `@Pattern`, etc.) declaratively express constraints that are checked automatically
- Validation at the controller boundary provides a clear, consistent error response (400 Bad Request) before any side effects occur
- Manual null checks and format validation scattered through service code is error-prone and inconsistent
- This is a defence-in-depth measure complementing client-side validation

## What good looks like

```java
@RestController
@RequestMapping("/api/users")
public class UserController {

    @PostMapping
    public ResponseEntity<UserDto> createUser(@Valid @RequestBody CreateUserRequest request) {
        // request is guaranteed to pass validation constraints
        return ResponseEntity.ok(userService.create(request));
    }
}

public class CreateUserRequest {

    @NotBlank(message = "Email is required")
    @Email(message = "Must be a valid email address")
    private String email;

    @NotBlank(message = "Name is required")
    @Size(min = 1, max = 255, message = "Name must be between 1 and 255 characters")
    private String name;
}
```

**Avoid:**

```java
@PostMapping
public ResponseEntity<UserDto> createUser(@RequestBody CreateUserRequest request) {
    // No @Valid — constraints on CreateUserRequest are never checked
    // Malformed input reaches business logic
    return ResponseEntity.ok(userService.create(request));
}
```

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Peer review** | Reviewers verify that `@RequestBody` parameters have `@Valid` or `@Validated` | Every PR |

**Primary enforcement: Peer review.** Reviewers should check all controller methods accepting request bodies. Optional: configure ArchUnit to enforce that all `@RequestBody` parameters are also annotated with `@Valid`.

## Quick reference

```java
@PostMapping
public ResponseEntity<Dto> create(@Valid @RequestBody CreateRequest request) {
    return ResponseEntity.ok(service.create(request));
}
```
