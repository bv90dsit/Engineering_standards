# JV-005: Spring Security configuration required for all web services

## Standard

Any Spring Boot web service MUST include `spring-boot-starter-security` as a dependency and MUST have an explicit `SecurityFilterChain` configuration. Unauthenticated endpoints MUST be explicitly whitelisted — the default posture is "deny all".

## Rationale

- Without Spring Security, all endpoints are unauthenticated by default — this is insecure-by-default
- Explicit security configuration forces teams to consciously decide which endpoints are public
- A deny-all default means new endpoints are automatically protected; forgetting to configure security for a new endpoint does not create a vulnerability
- Spring Security provides CSRF protection, session fixation protection, security headers, and authentication/authorisation in a single dependency
- This aligns with the NCSC principle of secure by default and defence in depth

## What good looks like

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/health", "/info").permitAll()  // explicitly whitelisted
                .anyRequest().authenticated()                      // deny-all default
            )
            .oauth2ResourceServer(oauth2 -> oauth2.jwt(Customizer.withDefaults()));

        return http.build();
    }
}
```

**build.gradle:**

```groovy
implementation 'org.springframework.boot:spring-boot-starter-security'
```

**Avoid:**

- Omitting `spring-boot-starter-security` entirely (all endpoints open)
- Using `.anyRequest().permitAll()` (effectively disabling security)
- Excluding Spring Security auto-configuration via `@SpringBootApplication(exclude = SecurityAutoConfiguration.class)`

## Enforcement

| Mechanism | What is checked | When |
|-----------|----------------|------|
| **Periodic audit** | Verify that all web services include `spring-boot-starter-security` and have a `SecurityFilterChain` bean | Quarterly / on-demand |

**Primary enforcement: Periodic audit.** Security reviews check that every deployed web service has an explicit security configuration. The audit verifies the dependency is present, auto-configuration is not excluded, and no blanket `permitAll()` is applied.
