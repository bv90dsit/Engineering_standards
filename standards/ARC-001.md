# ARC-001: Cloud-first architecture

## Standard

New services SHOULD be designed for public cloud deployment. Teams SHOULD use managed services over self-hosted infrastructure where they meet security and data classification requirements. Infrastructure SHOULD be defined as code.

## Rationale

- Cloud-first is UK Government policy (since 2013, reaffirmed in Technology Code of Practice)
- Managed services reduce operational burden and improve reliability
- Infrastructure-as-code enables repeatability, auditability, and disaster recovery
- Avoids capital expenditure and long procurement cycles

## What good looks like

- Services deployed to an approved public cloud (AWS, Azure, GCP via Crown Commercial frameworks)
- Infrastructure defined in Terraform, Pulumi, or CloudFormation — never manual console changes
- Managed databases, message queues, and compute (RDS, SQS, Cloud Run, etc.) over self-hosted
- Multi-AZ / multi-region for critical services
- Clear cost allocation and tagging strategy
- Exit strategy documented to avoid vendor lock-in

## When on-premises is acceptable

- Data classified above OFFICIAL-SENSITIVE with no approved cloud hosting
- Legacy systems mid-migration with a documented cloud migration roadmap
- Specific latency or data residency requirements that cloud cannot meet
